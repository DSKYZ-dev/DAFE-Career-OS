# Mode: autonomous — Autonomous Entry-Level / Low-Barrier Application Loop

> **Purpose**: When the candidate needs a position — any position — and wants maximum volume with quality. Finds, evaluates, tailors, and auto-applies to entry-level, minimal-requirement, "any-background" roles regardless of primary focus tracks.

## Philosophy

Standard pipeline optimizes for **fit** (score ≥ 4.0). Autonomous mode optimizes for **access** — it targets roles where:
- Requirements are minimal (0-2 years, "will train", "no experience needed")
- Competition is lower (less glamorous titles, non-tech companies hiring tech)
- Volume is high (graduate programs, rotational programs, apprenticeships)
- The candidate's *actual* skills exceed the stated bar

**Quality guardrails remain**: Every application gets a fully tailored CV + cover letter generated by LLM. No template spam.

| Situation | Use |
|-----------|-----|

## Invocation

```bash
# Dry run (fill forms, review in browser, no submit)
node autonomous-loop.mjs --max 20

# Actually submit (AUTO-SUBMIT - use with awareness)
node autonomous-loop.mjs --max 20 --auto-submit

# Run for 4 hours max, process up to 50 jobs
node autonomous-loop.mjs --max 50 --hours 4 --auto-submit

# Verbose dry run for debugging
node autonomous-loop.mjs --dry-run --verbose
```

### Options

| Flag | Default | Description |
|------|---------|-------------|
| `--max N` | 20 | Maximum jobs to process this run |
| `--auto-submit` | false | **Actually click Submit** on applications. Without this, fills forms and pauses for review. |
| `--hours N` | ∞ | Max wall-clock hours to run |
| `--dry-run` | false | Alias for no `--auto-submit` |
| `--verbose` | false | Detailed per-step logging |

## How It Works (Step by Step)

### 1. Scan & Enrich
Runs `scan.mjs` + `web-search.mjs` to populate `data/pipeline.md` with fresh postings from all configured portals.

### 2. Entry-Level Filter
For each pending pipeline entry, fetches the job page and checks:
- **Title keywords**: `entry level`, `junior`, `graduate`, `trainee`, `intern`, `apprentice`, `associate`, `analyst i`, `engineer i`, `new grad`, `0-1 year`, `0-2 years`, `willing to train`, `training provided`, `career starter`, `early career`, `fellowship`, `rotation program`, `leadership development`, `management trainee`
- **Description keywords**: Same list, scanned in full JD text
- **Exclusion**: Skips if senior keywords present (`senior`, `lead`, `principal`, `staff`, `architect`, `manager`, `director`, `5+ years`, `7+ years`, `10+ years`, `extensive experience`, `deep expertise`)

### 3. Permissive Evaluation
Runs full `cloud-eval.mjs` evaluation (A-G framework + legitimacy check) but uses a **lower threshold** (default 2.5/5, configurable via `scoreThreshold` in settings/dashboard).

> **Why not skip evaluation?** Legitimacy detection (Block G) still matters — ghost jobs waste time. The evaluation also provides the JD analysis needed for tailoring.

### 4. LLM-Tailored CV Generation
For each job passing threshold:
- Reads base `cv.md` + full JD text
- Calls LLM (via `cloud-eval.mjs` prompt) to **rewrite the CV** for this specific role
- **Preserves all facts** — only reorders, rephrases, mirrors JD terminology
- Output: tailored markdown → PDF via `generate-cv.mjs --tailor`

### 5. LLM-Tailored Cover Letter Generation
- Same LLM call with specialized prompt
- 3-4 paragraphs: hook → 2-3 matched achievements → company knowledge + culture fit → close
- Returns structured JSON payload → PDF via `generate-cover-letter.mjs --tailor`

### 6. Auto-Apply
Launches headless Playwright, navigates to apply URL:
- Detects ATS (Greenhouse, Lever, Workday, Ashby, SmartRecruiters, iCIMS, Taleo, BambooHR, generic)
- Fills all fields via label-text discovery + known selectors
- Uploads tailored CV + cover letter PDFs
- **If `--auto-submit`**: clicks Submit button, waits for confirmation
- **If not**: pauses browser for manual review (visible mode available)

### 7. Tracker Logging
On successful submit:
- Writes TSV entry to `batch/tracker-additions/`
- Saves full evaluation report to `reports/`
- Updates `data/applications.md` on next `merge-tracker.mjs` run

## Quality Guarantees (Non-Negotiable)

| Guarantee | Implementation |
|-----------|----------------|
| **No fabricated facts** | LLM prompts explicitly forbid invention; CV/CL use only base CV + public company info |
| **No template spam** | Every document is generated fresh per job via LLM |
| **Legitimacy check** | Block G (posting legitimacy) runs on every job — ghost jobs filtered |
| **Rate limiting** | Randomized delays (8-20s default) between applications; configurable |
| **Audit trail** | Every step logged to `logs/autonomous-YYYY-MM-DD.log` + tracker TSV |
| **User review possible** | Run without `--auto-submit` to inspect filled forms before submitting |

## Safety & Ethics

### The "No Spray & Pray" Rule
Autonomous mode **does not** apply to every job indiscriminately. It:
1. Filters for genuinely low-barrier roles
2. Evaluates each for legitimacy
3. Generates unique, tailored documents
4. Respects rate limits

### When NOT to Use --auto-submit
- You haven't reviewed a sample of generated CVs/CLs first
- Applying to roles requiring security clearance, government, or regulated industries
- Company has explicit "no automated applications" policy (rare but exists)
- You want to personally customize the "why this company" paragraph

### Recommended Workflow
1. Run `--dry-run --verbose --max 5` → inspect generated PDFs in `output/`
2. If quality is good, run `--max 20 --auto-submit`
3. Monitor `logs/autonomous-*.log` for any errors
4. Run `node merge-tracker.mjs` periodically to update main tracker

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| "0 entry-level jobs found" | Lower `scoreThreshold` in settings; check scan ran recently; broaden portals.yml |
| CV tailoring fails | Check `cloud-eval.mjs` API keys; LLM may be rate-limited |
| Playwright timeout on apply | Increase `--timeout` in auto-apply.mjs; some ATS are slow |
| Cover letter JSON parse error | LLM returned malformed JSON; check `logs/autonomous-*.log` for raw response |
| Duplicate applications | Run `node dedup-tracker.mjs`; autonomous mode checks existing tracker scores |
